    Introduction

    This API is a way to interact with your AdCoin Wallet without using the provided interface

    We have language bindings in Shell, and PHP! You can view code examples in the dark area to the right, and you can switch the programming language of the examples with the tabs in the top right.

    Authentication

    To authorize, use this code:

    <?php

$headers = [
  'X-AUTH-TOKEN: myApiKey',
];

$curl = curl_init("api_endpoint_here");

curl_setopt($curl, CURLOPT_HTTPHEADER, $headers);
curl_setopt($curl, CURLOPT_RETURNTRANSFER, true);

$result = json_decode(curl_exec($curl));

return $result;

    curl "api_endpoint_here"
  -H "X-AUTH-TOKEN: myApiKey"

    Make sure to replace myApiKey with your API key.

    The API uses API keys to allow access to the API. You can register your API key in the Wallet, login/register open the menu on the top right and select API key

    The API expects the API key to be included in all API requests to the server in a header that looks like the following:

    X-AUTH-TOKEN: myApiKey

    You must replace myApiKey with your personal API key.

    Status codes

    Status Code Meaning
    200 Success -- Ready to go!
    201 Created -- The resource is created
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    404 Not Found -- The specified resource could not be found.
    405 Method Not Allowed -- You used the wrong HTTP method
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.

    Pay AdCoin

    curl "https://wallet.getadcoin.com/api/payto"
  -X POST
  -H "X-AUTH-TOKEN: myApiKey"
  -F "amount=100"
  -F "email=john@doe.com"

    <?php

$headers = [
  'X-AUTH-TOKEN: myApiKey',
];

$curl = curl_init("https://wallet.getadcoin.com/api/payto");

curl_setopt($curl, CURLOPT_HTTPHEADER, $headers);
curl_setopt($curl, CURLOPT_RETURNTRANSFER, true);
curl_setopt($curl, CURLOPT_POST, true);

$parameters = [
  'amount' => 100,
  'email' => 'john@doe.com',
];

curl_setopt($curl, CURLOPT_POSTFIELDS, http_build_query($parameters));

$result = json_decode(curl_exec($curl));

return $result;

    Example response:

    {
  "transaction": "461ee445734f4fac4a367131b3e6275e7ce7df9fcccc3281edcf2703e8d5a9b9"
}

    This endpoint can be used to pay AdCoin to a email or address.

    Only one of email or address should be given

    HTTP Request

    POST https://wallet.getadcoin.com/api/payto

    URL Parameters

    Parameter Description
    amount The amount in AdCoin
    email E-mail address to pay AdCoin to
    address AdCoin address to pay AdCoin to

    Create payment

    curl "https://wallet.getadcoin.com/api/payments"
  -X POST
  -H "X-AUTH-TOKEN: myApiKey"
  -F "amount=100"
  -F "redirectUrl=https://myapp.com/checkout/status"
  -F "webhookUrl=https://myapp.com/checkout/webhook"
  -F "description='500 advertisement credits'"
  -F "metadata={\"name\": \"TestUser\"}"

    <?php

$headers = [
  'X-AUTH-TOKEN: myApiKey',
];

$curl = curl_init("https://wallet.getadcoin.com/api/payments");

curl_setopt($curl, CURLOPT_HTTPHEADER, $headers);
curl_setopt($curl, CURLOPT_RETURNTRANSFER, true);
curl_setopt($curl, CURLOPT_POST, true);

$parameters = [
  'amount' => 100,
  'redirectUrl' => 'https://myapp.com/checkout/status',
  'webhookUrl' => 'https://myapp.com/checkout/webhook',
  'description' => '500 advertisement credits',
  'metadata' => json_encode(['name' => 'TestUser']),
];

curl_setopt($curl, CURLOPT_POSTFIELDS, http_build_query($parameters));

$result = json_decode(curl_exec($curl));

return $result;

    Example response:

  "id": "b43292ea-0f18-11e8-8dd3-0242ac130003",

    This endpoint creates a payment.

    HTTP Request

    POST https://wallet.getadcoin.com/api/payments

    URL Parameters

    Parameter Description
    amount The amount in AdCoin
    redirectUrl Where the customer should be redirect after succesful payment
    webhookUrl The URL that gets called when the payment status changes
    description Description that is shown to the user at the payment page
    metadata Metadata for your own reference, included in all further responses

    This endpoint creates a order.

    HTTP Request

    POST https://wallet.getadcoin.com/api/payments

    URL Parameters

    Parameter Description
    amount The amount in euro
    paymentMethod One of the paymentmethods returned by /paymentmethods
    idealIssuer iDeal issuer, found in /paymentmethods (Only required if paymentMethod is iDeal)
    coin Coin to use, found in /paymentmethods (Only required if paymentMethod is cryptocurrency)
